Roadmap

From v1.0 freeze to v2.0 royalty-free open standard.

Three milestones: v1.0 (April 2026, shipped), v1.1 (Q3 2026, sector overlays + KYE Conformant™ badge issuance), v2.0 (2027, federation v2 + patent-track to royalty-free open standard).

Now · v1.0 (April 2026)

Shipped: the open contract.

  • 36 v1.0 normative profiles + 5 v1.1 preview — Core, Gateway, Payments, Federation, Credentials, Attestation, Signals, Transparency, Conformance, Treasury, Custody, Healthcare, Telemetry, Capability, Recovery + 26 sector overlays.
  • 193 OpenAPI operations across 87+ runtime endpoints; 57 JSON Schemas with 60 validated examples.
  • Reference Gateway (TypeScript), embedded PDP library, PEP middleware (Express).
  • Three SDKs — TypeScript, Python, Go — and 38 conformance fixtures (38/38 pass).
  • 266 control mappings across SOC 2, ISO 27001:2022, PCI DSS 4.0, PSD2/PSD3, DORA, NIS2, EU AI Act, NIST 800-207, HIPAA.
Next · v1.1 (Q3 2026)

Sector overlays + conformance certification programme.

  • Healthcare clinical / payer / research overlays (kye-healthcare-clinical-1.0, kye-healthcare-payer-1.0, kye-healthcare-research-1.0) + 42 CFR Part 2 alignment.
  • Extended signal-bus durability options (at-least-once with idempotency, exactly-once via outbox).
  • conformance-report.json + conformance-fixture.json schemas finalised.
  • KYE Conformant™ badge issuance via the public registry; vendor self-attestation portal.
  • Independent test-vector runners (community-contributed).
Later · v2.0 (2027)

Federation v2 + royalty-free open standard.

  • Federation v2 with multi-hop attenuation and cross-jurisdiction proofs.
  • Patent-track algorithms moved to royalty-free open standard (Linux Foundation / OpenWallet Foundation governance).
  • Post-quantum cryptography overlay (algorithm choice deferred to v2.0 RFC).
  • Native mobile SDKs (iOS / Android) for agent-side workloads.

For specific feature requests or RFC discussion, see github.com/KYE-Protocol/Discussions.
