Step 2 · Taxonomies

Parent / child classification of KYE™ names.

Dictionaries (step 1) define the names KYE™ recognises. Taxonomies (step 2) organise those names into parent / child hierarchies so a sector profile, regulator or auditor can talk about a class without enumerating every member.

1 · What lives here

Sixteen v1.0 canonical taxonomies.

KYE Taxonomy & Metadata Profile™ ships sixteen canonical taxonomies covering entity classes, capability families, side-effect levels, data classes, decision codes, lifecycle states, jurisdictions, sectors, profile families, certification tiers, risk tiers, evidence kinds, signal families, action families, redaction classes and oversight roles. Each taxonomy is a published, versioned, hash-bound artefact.

2 · Why a taxonomy is not an ontology

Hierarchy, not meaning.

A taxonomy organises terms. An ontology defines how terms relate. The taxonomy says "card_purchase is a kind of payment_initiation". The ontology says "card_purchase requires an authority_grant and is constrained by an amount_limit." KYE™ needs both, in that order.
